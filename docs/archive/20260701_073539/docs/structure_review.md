# 구조 리뷰 문서

이 문서는 과거 채점 구조 검토 내용을 보존하기 위한 문서이다.

현재 프로젝트의 기준 구조는 다음 문서를 따른다.

- `docs/grading_architecture.md`

과거에는 답안을 다음 4개 블록으로 나누는 방식을 검토했다.

1. 배경
2. 문제점
3. Fact 기반 문제점 설명
4. 기술사적 의견과 현실적 대책

현재는 이 구조를 직접 채점 구조로 사용하지 않는다.

현재 채점 구조는 A/B/C/D/E 25점 구조이다.

| 항목 | 설명 | 배점 |
|---|---|---:|
| A | 배경과 문제 진입 | 4 |
| B | 문제 요구 파악 | 5 |
| C | 유형별 Fact 기반 내용 설명 | 8 |
| D | 현장 적용·설계 판단·제언 | 6 |
| E | 연결성·면접 방어 가능성 | 2 |

따라서 이 문서는 참고용이며, 실제 구현 기준은 `docs/grading_architecture.md`를 따른다.
