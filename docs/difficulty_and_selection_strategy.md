# 난이도와 문항 선택 전략 설계

이 문서는 산업계측제어기술사 답안 채점 Bot에서 문제 난이도와 문항 선택 전략을 어떻게 반영할지 정의한다.

기존 A/B/C/D/E 25점 채점 구조는 유지한다. 난이도와 문항 선택 중요도는 기존 점수 체계를 대체하지 않고, 채점 엄격도와 시험 전략 평가를 보완하는 layer로 사용한다.

## 1. 핵심 문제의식

산업계측제어기술사 문제는 모든 주제가 같은 깊이로 출제되지 않는다.

일부 문제는 정의, 구성, 장단점, 현장 적용 수준에서 출제된다. 반면 제어이론, feedback system, 2차 시스템, 안정도, 과도응답과 같은 주제는 이론적 깊이가 크고 반복적으로 출제되는 핵심 변별 영역이다.

따라서 쉬운 문제 4개를 안정적으로 선택하는 전략과, 제어이론 문제를 포함하여 고득점 band를 노리는 전략은 1:1로 비교하면 안 된다.

## 2. 최종 원칙

난이도와 선택 중요도는 분리한다.

| 구분 | 의미 | 적용 위치 |
|---|---|---|
| Answer Difficulty | 선택한 문제 자체의 난이도와 채점 엄격도 | 개별 답안 채점 |
| Selection Importance | 6문제 중 반드시 준비해야 할 핵심 문항인지 여부 | 문항 선택 전략 평가 |

핵심 원칙은 다음과 같다.

- 쉬운 문제는 안정 점수형이지만 고득점 ceiling이 낮다.
- 제어이론 문제는 고위험·고보상형이다.
- 제어이론을 선택했다는 사실만으로 가산점은 주지 않는다.
- 제어이론을 정확히 풀었을 때만 상위 점수 band가 열린다.
- 제어이론을 틀리면 fatal error cap을 강하게 적용한다.
- 제어이론을 회피하면 개별 답안 점수를 깎지 않고 선택 전략 risk로 표시한다.

## 3. Answer Difficulty

| Profile | 의미 | 대표 주제 | 기본 ceiling |
|---|---|---|---:|
| BASIC_CONCEPT | 정의, 개념, 구성, 적용 범위 중심 | 용어 정의, 기본 개념 | 19 |
| FIELD_APPLICATION | 현장 적용, 선정, 문제점, 개선방안 중심 | 밸브, 센서, 교정, 캐비테이션 | 21 |
| DESIGN_EVALUATION | 설계, 평가, 효과분석, 지표 중심 | SIL, OEE, ROI, 시스템 평가 | 22 |
| THEORY_CORE | 수식, 동특성, 안정도, 응답특성 중심 | 제어이론, 2차 시스템, feedback | 25 |

THEORY_CORE는 기본 ceiling이 가장 높다. 단, 상위 band는 자동으로 열리지 않는다. 표준 모델, 핵심 변수, 수식 관계, 응답특성, 현장 의미를 정확히 연결했을 때만 21점 이상 고득점 band가 열린다.

## 4. 쉬운 문제와 제어이론 문제의 1:1 비교

쉬운 문제:

- 안정적으로 14~17점 확보 가능
- 좋은 답안이면 18~19점 가능
- 매우 잘 써도 고득점 ceiling이 낮음
- 치명 오류 가능성은 낮음

제어이론 문제:

- 틀리면 3~10점까지 낮아질 수 있음
- 부분적으로 쓰면 11~15점 수준
- 표준 모델과 응답특성을 정확히 쓰면 17~20점 가능
- 수식, 물리 의미, 현장 tuning 판단까지 연결하면 21~25점 가능

따라서 제어이론 문제는 “선택 가산점”이 아니라 “고득점 band unlock” 구조로 관리한다.

## 5. Score Band Unlock

THEORY_CORE 문제의 점수 band는 다음과 같이 본다.

| 답안 상태 | 점수 band |
|---|---:|
| 핵심 수식·모델 없음 | 3~10 |
| 표준 모델 일부 언급, 변수·응답특성 부족 | 9~14 |
| 표준 모델, 감쇠비, 고유진동수, 응답특성 대체로 정확 | 15~20 |
| 수식, 극점, 응답특성, 현장 tuning 판단까지 연결 | 21~25 |

BASIC_CONCEPT 문제의 점수 band는 다음과 같이 본다.

| 답안 상태 | 점수 band |
|---|---:|
| 정의만 있음 | 5~10 |
| 정의와 구성 설명 | 11~15 |
| 장단점과 적용 범위 포함 | 15~17 |
| 현장 적용과 한계까지 포함 | 18~19 |

따라서 쉬운 문제 4개로 평균 통과는 가능하지만, 고득점 안정성은 낮다. 반대로 제어이론 문제는 정확히 풀면 고득점 가능성이 열리지만, 핵심 오류가 있으면 위험하다.

## 6. Selection Importance

| Importance | 의미 |
|---|---|
| CORE_MUST_PREPARE | 반복 출제되는 핵심 변별 주제. 반드시 준비해야 함 |
| HIGH_PRIORITY | 선택 우선순위가 높은 주제 |
| NORMAL | 일반 선택 가능 주제 |
| OPTIONAL | 안전하지만 고득점 ceiling이 낮을 수 있는 주제 |

제어이론, feedback system, 2차 시스템, 안정도, 과도응답 계열은 CORE_MUST_PREPARE로 관리한다.

## 7. 선택 정책

| Policy | 의미 |
|---|---|
| attempt_if_floor_met | 최소 득점 가능성이 있으면 선택 권장 |
| safe_choice | 안정 점수 확보용 선택 |
| balanced_choice | 난이도와 안정성을 균형 있게 고려 |
| avoid_if_fatal_risk | 핵심 오류 가능성이 높으면 실전 선택 위험 |

2차 시스템은 다음 정책을 가진다.

- difficulty: THEORY_CORE
- importance: CORE_MUST_PREPARE
- selection_policy: attempt_if_floor_met
- minimum_attempt_floor: 12
- target_score: 15
- excellent_score_band: 21~25
- omission_risk: high
- fatal_error_risk: high

의미는 다음과 같다.

2차 시스템은 반드시 준비해야 한다. 시험에 나오면 선택 우선순위가 높다. 다만 최소 12점 수준의 답안을 쓸 자신이 없으면, 실전에서는 선택이 오히려 위험할 수 있다.

## 8. 시나리오 판단

### A. 제어이론 회피 + 쉬운 문제 4개 선택 + 평균 통과

판단:

- 답안 점수: 통과 가능
- 선택 전략: 안정형이지만 위험
- 고득점 가능성: 낮음
- 핵심 변별 주제 대응력: 부족

출력 예:

선택한 문제들은 안정적이지만 고득점 ceiling이 낮습니다. 제어이론 핵심 반복 문항을 회피했으므로, 고득점 안정성은 낮게 평가합니다.

### B. 제어이론 선택 + 제어이론 감점 + 나머지 3문제 부족

판단:

- 답안 점수: 낮음
- 선택 전략 방향: 타당
- 핵심 주제 실행력: 부족
- 결과: 불합격 가능

출력 예:

제어이론 핵심 문항을 선택한 방향은 타당합니다. 그러나 THEORY_CORE 문제는 정확성이 핵심이므로, 표준 모델·감쇠비·극점·응답특성 설명이 부족하면 고득점으로 인정할 수 없습니다.

## 9. 시스템 반영 방식

한 문제 채점에서는 다음만 표시한다.

- 문제 유형
- 난이도 Profile
- 선택 중요도
- 점수 ceiling 또는 unlock 조건
- fatal error risk

6문제 세트 평가에서는 다음을 평가한다.

- CORE_MUST_PREPARE 주제가 출제되었는가
- 해당 주제를 선택했는가
- 선택한 4문제의 ceiling 조합이 어떤가
- 쉬운 문제만 골라 고득점 ceiling이 낮아졌는가
- 제어이론을 선택했을 때 예상 최소점이 floor를 넘는가

개별 답안 점수와 선택 전략 평가는 분리한다.

## 10. 최종 원칙

제어이론은 “무조건 고르면 점수를 더 주는 문제”가 아니다.

제어이론은 다음과 같이 관리한다.

- 준비 단계: 반드시 준비
- 실전 선택: 최소 득점 가능하면 선택
- 채점 단계: 정확성 중심으로 엄격 채점
- 고득점: 정확히 풀었을 때만 상위 band unlock
- 전략 평가: 회피 시 고득점 안정성 낮음 표시

쉬운 문제는 다음과 같이 관리한다.

- 안정 점수 확보 가능
- 답안이 좋아도 ceiling 존재
- 고득점 변별력은 낮음
- 제어이론 회피 전략의 대체재는 될 수 있으나, 고득점 전략은 아님
