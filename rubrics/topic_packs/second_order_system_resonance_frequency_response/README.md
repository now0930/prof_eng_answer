# 2차 시스템 주파수응답과 공진주파수 Topic Pack

## 목적

이 topic pack은 “표준 2차 시스템의 주파수응답과 공진주파수를 설명하시오.” 유형의 답안을 평가하기 위한 기준이다.

이 topic의 핵심은 시간응답의 감쇠비별 분류가 아니라, 주파수응답에서 공진 peak가 언제 발생하고 공진주파수 `ωr`가 어떤 조건에서 정의되는지를 설명하는 것이다.

---

## 대표 문제

- 표준 2차 시스템의 주파수응답과 공진주파수를 설명하시오.
- 2차계에서 감쇠비와 공진주파수의 관계를 설명하시오.
- 시간응답 overshoot와 주파수응답 resonance의 차이를 설명하시오.
- 2차 시스템에서 감쇠비가 resonant peak와 bandwidth에 미치는 영향을 설명하시오.

---

## 핵심 정답 기준

- 표준 2차 시스템 전달함수는 보통 다음과 같이 표현한다.
  - `G(s) = Kωn² / (s² + 2ζωn s + ωn²)`
- 주파수응답은 `s = jω`를 대입하여 해석한다.
- 정규화 주파수비는 `r = ω/ωn`이다.
- 정규화 magnitude는 다음 형태로 설명할 수 있다.
  - `|G(jω)| / K = 1 / sqrt((1-r²)² + (2ζr)²)`
- 공진 peak는 감쇠비가 충분히 작을 때 발생한다.
- 공진주파수는 다음 조건에서 정의된다.
  - `ωr = ωn sqrt(1 - 2ζ²)`
  - 존재 조건: `0 ≤ ζ < 1/sqrt(2)`
- `ζ ≥ 1/sqrt(2)`이면 magnitude가 단조 감소하여 뚜렷한 공진 peak가 없다.
- resonant peak는 다음과 같이 표현할 수 있다.
  - `Mr = 1 / (2ζ sqrt(1-ζ²))`
  - 일반적으로 `0 < ζ < 1/sqrt(2)`에서 유한한 peak로 다룬다.
- 감쇠비가 증가하면 공진 peak는 감소하고 응답은 완만해진다.
- 시간응답 overshoot와 주파수응답 resonance는 관련은 있지만 같은 개념이 아니다.
  - overshoot 조건: 일반적으로 `0 < ζ < 1`
  - resonance peak 조건: `0 < ζ < 1/sqrt(2)`
- `ωd = ωn sqrt(1-ζ²)`는 시간응답의 감쇠진동수이고, `ωr = ωn sqrt(1-2ζ²)`는 주파수응답의 공진주파수이다.
- 따라서 `ωd`와 `ωr`를 혼동하면 안 된다.

---

## 정답으로 인정할 표현

- `ζ < 1/√2`에서 공진 peak가 존재한다.
- `ζ ≥ 1/√2`에서는 공진 peak가 나타나지 않는다.
- `ωr = ωn√(1-2ζ²)`는 공진주파수이다.
- `ωd = ωn√(1-ζ²)`는 감쇠진동수이며 공진주파수와 다르다.
- 감쇠비가 작을수록 resonant peak가 커진다.
- 감쇠비가 커질수록 peak는 낮아지고 주파수응답은 완만해진다.
- 시간영역 overshoot와 주파수영역 resonance는 구분해야 한다.
- 공진 peak는 기계 진동, 센서 동특성, 제어루프 bandwidth, 노이즈 증폭, 설비 보호와 관련된다.

---

## 핵심 fatal 오류

- `ζ = 1`을 공진 조건 또는 공진주파수 조건으로 설명
- `ωr = ωn√(1-ζ²)`라고 설명하여 감쇠진동수 `ωd`와 공진주파수 `ωr`를 혼동
- `ωd`와 `ωr`를 같은 값 또는 같은 개념으로 설명
- `ζ ≥ 1/√2`에서도 공진 peak가 존재한다고 설명
- 모든 감쇠비에서 공진주파수가 존재한다고 설명
- `ωr = ωn`이 항상 성립한다고 설명
- 시간응답 overshoot 조건 `0 < ζ < 1`을 공진 peak 조건으로 그대로 사용
- `ζ = 1/√2`를 임계감쇠로 설명
- `ζ = 1` 임계감쇠와 `ζ = 1/√2` 공진 peak 한계 조건을 혼동
- resonance frequency를 natural frequency와 항상 동일하다고 설명

---

## warn 수준의 부족한 표현

- 공진주파수 공식은 썼지만 존재 조건 `ζ < 1/√2`를 빠뜨림
- 공진 peak가 감소한다고만 쓰고 magnitude 식 또는 주파수비 설명이 없음
- 시간응답 overshoot와 주파수응답 resonance를 구분하지 않음
- 현장 적용에서 기계 공진, 센서 노이즈, bandwidth, 제어루프 튜닝과 연결하지 않음
- `ωn`, `ωr`, `ωd`의 의미를 구분하지 않음
- 수식만 쓰고 감쇠비 증가에 따른 응답 해석이 부족함

---

## false positive 주의사항

- `ωr ≈ ωn`은 감쇠비가 매우 작을 때의 근사 설명이면 허용 가능하다.
- `ζ = 1`을 시간응답의 임계감쇠로 설명하는 것은 정상이다.
- `ζ = 1`이 공진 조건이라고 설명할 때만 오류이다.
- `ζ = 1/√2`는 공진 peak가 사라지는 경계 조건이지 임계감쇠가 아니다.
- `ωd = ωn√(1-ζ²)`는 시간응답 감쇠진동수 설명으로 쓰면 정상이다.
- `ωd`를 공진주파수라고 부르거나 `ωr`와 동일시하면 오류이다.
- overshoot와 resonance의 관련성을 언급하는 것은 정상이나, 두 조건을 동일시하면 오류이다.

## 표와 다이어그램 처리 메모

표, Bode plot sketch, magnitude curve, 주파수응답 곡선, pole diagram은 그 자체가 정답 또는 오답이 아니다. 거기서 읽히는 claim을 평가한다.

### Fact Anchor에서 evidence로 인정할 수 있는 claim

- magnitude 식 또는 정규화 주파수비 `r = ω/ωn`
- `ωr = ωn√(1-2ζ²)`
- `ζ < 1/√2` 존재 조건
- 감쇠비 증가에 따른 peak 감소
- `ωn`, `ωr`, `ωd` 구분
- 시간응답 overshoot와 주파수응답 resonance 구분
- 기계 공진, 센서 동특성, 제어루프 bandwidth, 노이즈 증폭과의 연결

### Logic Check에서 fatal 또는 major 후보로 볼 수 있는 claim

- 표에서 `ζ = 1 → resonance`라고 분류
- 표에서 `ζ = 1/√2 → critical damping`이라고 분류
- 그래프 설명에서 모든 `ζ`에서 peak가 있다고 설명
- 수식에서 `ωr = ωn√(1-ζ²)`를 공진주파수로 제시
- `ωd = ωr`라고 설명
- `ωr = ωn`이 항상 성립한다고 설명
- overshoot 조건과 resonance 조건을 동일하게 설명

그림 위치만으로 fatal 처리하지 않는다. 그래프 라벨, 수식, 본문 설명, 표의 mapping이 함께 정답과 충돌할 때 Logic Check 대상으로 본다.
